# Contributing to Glacier.Vector

First off, thank you for considering contributing to Glacier.Vector! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct.

## How Can I Contribute?

### Reporting Bugs
*   **Check the existing issues** to see if the bug has already been reported.
*   If you can't find an open issue addressing the problem, **open a new one**.
*   Be as detailed as possible:
    *   What were you doing?
    *   What was the expected output?
    *   What was the actual output?
    *   Provide a minimal reproducible example (code snippet).

### Suggesting Enhancements
*   Open an issue with the tag `enhancement`.
*   Explain the use case and how it would benefit the project.

### Pull Requests
1.  Fork the repository.
2.  Create a new branch (`git checkout -b feature/AmazingFeature`).
3.  Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4.  Push to the branch (`git push origin feature/AmazingFeature`).
5.  Open a Pull Request.

## Development Setup

1.  Install the **.NET 10 SDK**.
2.  Clone the repository.
3.  Open `Glacier.Vector.slnx` in Visual Studio 2022 or use the .NET CLI.
4.  Run tests: `dotnet test`.

## Style Guide
*   Use standard C# coding conventions.
*   Ensure all public methods have XML documentation.
*   Performance-critical code should be benchmarked using BenchmarkDotNet (if applicable).

## Questions?
Feel free to open an issue with the tag `question`.
